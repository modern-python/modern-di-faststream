# Design: Canonical minimal-DI example (faststream pilot)

## Summary

Ship a canonical, runnable example in `modern-di-faststream`: a minimal
`examples/` app wiring one `Settings` (APP scope) and one `Service` (REQUEST
scope) into a subscriber via the real `FromDI` idiom, an in-memory smoke test
that proves it runs, and the README `Usage example:` line the audit found
missing. This flips the two coupled gaps the blessed-ready audit scored on this
repo — **D2 (canonical example) 1 -> 2** and **D6 (README consistency) 1 -> 2**.
faststream is the pilot: it locks the reusable shape that the other 9
`@inject`/`FromDI` integrations then copy, and that core
`docs/integrations/writing-integrations.md` codifies as an authoring rule during
rollout.

## Motivation

The [blessed-ready on-ramp audit](https://github.com/modern-python/modern-di/blob/main/planning/audits/2026-07-22-blessed-ready-onramp-report.md)
found 10 of 12 integrations at **D2=1 / D6=1**: a complete inline README example
but no *dedicated, linked* canonical example, so neither the `Usage example:`
line (D6) nor a D2=2 score is reachable. Only `fastapi` and `litestar` clear it,
by linking separate `*-sqlalchemy-template` repos. The audit ranked closing this
as the **top cross-cutting fix** (§4), and separately found **no integration
ships an `examples/` dir at all**. faststream is a stated adoption beachhead and
a Σ=9 close miss, so it is the pilot: get the shape right once, roll to nine.

## Design

**The example** — `examples/app.py` (+ `examples/__init__.py`): the smallest
runnable faststream app that demonstrates the recommended wiring, matching the
README snippet's shape but as a complete program:

- `Settings` — `providers.Factory(scope=Scope.APP, creator=Settings)`.
- `GreetingService` — `providers.Factory(scope=Scope.REQUEST, ...)`, depends on
  `Settings` by type (auto-injected).
- one `@broker.subscriber(...)` handler taking the service via
  `FromDI(Dependencies.service)`, wired by `setup_di(app, container)` on a
  `NatsBroker` app. A `# run: faststream run examples.app:app (needs a NATS
  broker)` header comment states the real-run requirement.

**Smoke test** — `tests/test_example.py`: imports the example's `app`/`broker`,
and using the repo's existing in-memory idiom (`TestNatsBroker` + `TestApp`,
already used in `tests/test_lifespan.py`) publishes one message and asserts the
injected `GreetingService` resolved. This proves the example runs with **no
infra**, and keeps it from rotting.

**Coverage** — add `[tool.coverage.run] omit = ["examples/*"]` to `pyproject.toml`
(the same move core uses for `benchmarks/*`), so `--cov-fail-under=100` holds
without covering broker-boot/`__main__` glue. Runnability is guaranteed by the
smoke test executing the wiring, not by line coverage of the boot code.

**README** — insert `Usage example: [examples/](./examples)` at line ~24 (the
exact D6 gap the audit cites, `README.md:22-24`), directly under the `Full guide`
line, matching `fastapi`/`litestar`'s placement. The existing inline `## Usage`
snippet stays — the example is its fuller, runnable form.

**The reusable pattern** (what rollout copies, per integration): `examples/app.py`
with the same Settings+Service+handler shape in that framework's idiom; a
`tests/` smoke test via the repo's own in-memory double; `omit = ["examples/*"]`;
and the README `Usage example:` line. Rollout codifies this in core
`docs/integrations/writing-integrations.md`.

No `modern_di_faststream/` source changes, so no `architecture/` promotion — this
adds documentation-grade example + test, not a capability change.

## Non-goals

- The other 9 integrations and the core `writing-integrations.md` codification —
  the rollout follow-up, which references this locked pattern. One line each.
- A realistic/DB starter — minimal DI demo only (the audit's chosen ambition).
- New runtime dependencies — the example uses only `faststream` + `modern-di`,
  already present.
- Editing the inline README snippet or any integration behavior.

## Testing

- `just test-ci` green: 100% coverage holds via the `omit` + the smoke test,
  which publishes a real message and asserts resolution (real behavior, not a
  mock).
- `just lint-ci` green (ruff/ty/eof/format over the new files).
- `just check-planning` green (this bundle validates).
- Manual sanity: `faststream run examples.app:app` starts against a local NATS
  broker (documented, not a CI gate).

## Risk

- **`omit` hides example rot** (medium x low) — mitigated: the smoke test
  imports and executes the example every CI run; a broken example fails the test.
- **Example drifts from the README snippet** (low x low) — they share one shape;
  keep the example the runnable superset of the snippet.
- **Pattern doesn't transfer to infra-heavy / `@inject` integrations** (low x
  medium) — the pilot deliberately uses the in-memory-test-double idiom every
  repo already has; rollout validates per repo, and codification captures the
  rule, not faststream-specific code.
