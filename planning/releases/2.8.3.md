# modern-di-faststream 2.8.3 — internal hardening, no behaviour change

A maintenance release from an architecture review. **No public API or behaviour
changes** — `setup_di`, `FromDI`, `fetch_di_container`, and
`faststream_message_provider` are unchanged.

## Internal refactors

- **Dropped a vestigial pass-through.** The `faststream_context` property on the
  DI middleware, left behind by the FastStream 0.7 migration, returned
  `self.context` unchanged. Inlined and removed (#18).
- **Named the container context keys.** The root and per-message containers are
  handed off through FastStream's `ContextRepo` under string keys; these are now
  module-level constants instead of bare literals duplicated across writer and
  reader, so a rename or typo is a `NameError` at import rather than a silent
  runtime miss (#19).

## Downstream

No action needed. No API change; version floors do not need bumping.

## Internals

- Recorded a decision to keep the `_DIMiddlewareFactory`/`_DiMiddleware` split —
  it adapts FastStream's middleware-construction contract (#20).
- Authored the first `architecture/` capability file documenting the DI seam.
- Test suite green at 100% coverage on Python 3.10–3.14 throughout.
