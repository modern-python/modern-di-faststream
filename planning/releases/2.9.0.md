# modern-di-faststream 2.9.0 — adopt modern-di's blessed integration seams

A maintenance release. **No public API or behaviour changes** — `setup_di`,
`FromDI`, `fetch_di_container`, and `faststream_message_provider` are
unchanged.

## Internal refactors

- **Registration and resolution now go through modern-di's blessed seams.**
  `container.providers_registry.add_providers(...)` (a reach-in) is replaced
  with `container.add_providers(...)`, and the `isinstance`-on-`AbstractProvider`
  dispatch in `Dependency.__call__` is replaced with
  `container.resolve_dependency(...)` — the provider-or-type entry point
  modern-di added for exactly this purpose (modern-python/modern-di#283).

## Packaging

- Bumped the `modern-di` floor to `>=2.25,<3` — the version that ships
  `add_providers` and `resolve_dependency`.

## Downstream

No action needed beyond the floor bump already applied. No API change.

## Internals

- Test suite green at 100% coverage on Python 3.10–3.14 throughout.
